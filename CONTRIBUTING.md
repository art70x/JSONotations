# Contributing to JSONotations

Thank you for your interest in contributing to JSONotations! We welcome contributions from the community.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project.
2. **Clone the Fork**: `git clone https://github.com/art70x/JSONotation.git`
3. **Create a Branch**: `git checkout -b feature/your-feature-name`
4. **Make Changes**: Implement your feature or fix.
5. **Lint Your Code**: Ensure your code passes the linting checks: `npm run lint`
6. **Submit a PR**: Open a Pull Request with a clear description of your changes.

## Development Guidelines

- **Clean Code**: Follow the existing code style (2-space indentation, functional components).
- **No Backend**: This project is strictly frontend-only. All persistence must use LocalStorage or browser-based APIs.
- **A11Y**: Ensure new features are accessible via keyboard and screen readers.
- **Components**: Reuse existing Shadcn UI components wherever possible.

## Reporting Issues

If you find a bug, please open an issue with:

- A clear description of the problem.
- Steps to reproduce the bug.
- Any relevant JSON data (anonymized).
- Browser and OS information.

We appreciate your help in making JSONotations the best JSON tool for developers!
