# Future Feature Ideas (Non-Backend)

This document tracks potential features and enhancements for JSONotations that can be implemented entirely on the frontend.

## 📥 Import / Export
- [ ] **Conversion**: Support converting JSON to/from YAML, XML, and TOML.
- [ ] **Advanced CSV Export**: Customizable delimiter and field selection.
- [ ] **Pasted URL Import**: Fetch JSON directly from a URL (CORS permitting).

## 🔍 Tools & Transformation
- [ ] **JSONPath Evaluator**: Real-time filtering and querying using JSONPath expressions.
- [ ] **JSON Patch Generator**: Generate RFC 6902 patches by comparing two JSON states.
- [ ] **Template Engine**: Support for basic data transformation templates (e.g., Handlebars).
- [ ] **Base64 Encoder/Decoder**: Inline tools for handling Base64 strings within JSON values.

## 🎨 UI / UX
- [ ] **Custom Themes**: Allow users to select different syntax highlighting themes.
- [ ] **PWA Support**: Full offline capabilities via Service Workers.
- [ ] **Multi-window Sync**: Sync editor state across browser tabs using BroadcastChannel API.
- [ ] **History Sidebar**: Visual timeline of recent edits (stored in LocalStorage).

## 🛠️ Editor Enhancements
- [ ] **Regex Search**: Support for regular expressions in the search bar.
- [ ] **Multi-cursor Support**: Integration of more advanced CodeMirror features.
- [ ] **Breadcrumb Navigation**: Path breadcrumbs at the top of the editor for better context.
